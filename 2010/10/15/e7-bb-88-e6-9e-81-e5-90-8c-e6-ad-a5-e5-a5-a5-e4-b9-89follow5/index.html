终极同步奥义——follow5

2010.10.15 Ivan 工作相关

前几天用月光的教程搭个GAE的应用，每天运行正常的很，流量也够猛，就是没有效果，今天再次研究才发现，原来是sina这鸟地方也跟风改成Oauth认证了，我说怎么就同步不过去

再次参阅教程改进，看来在同步到墙内只靠GAE是力不从心了，只能配合follow5来实现。这follow5听说许久，本以为就是个普普通通的微博服务，不过今天注册了一下，发现这其它地方虽然亮点有限，可这同步功能可算是我见过的最强大的，墙内各网，无所不能同，当然，除了狗日的腾讯

步骤还是如以前，大概记录一下：

1. 先建个GAE的app, 起个名当然

2. 下载脚本 http://j.mp/cabJi6， 没测试是不是和我当时下的一样，应该是这个， 原出处懒得去看了

3. 小改一下：

用写字板打开app.yaml，修改第一行的yourappid为你建立的APP应用名字，保存关闭

用写字板打开cron.yaml，修改schedule: every 5 minutes这一行的数字为你希望的同步间隔，保存关闭；

用写字板打开twitter.py，找到下面这段：

# You MUST modify your username and password here #

#ret = send_sina_msgs("username @sina.com"," password ",text)

#ret = send_163_msgs("username@163.com","password",text)

#ret = send_sohu_msgs("username@sohu.com","password",text)

#ret = send_9911_msgs("username","password",text)

#ret = send_zuosa_msgs("username "," password ",text)

#ret = send_renjian_msgs("username","password",text)

#ret = send_follow5_msgs("username "," password ",text)

#ret = send_pingfm_msgs("api_key","user_app_key",text)

#ret = send_hellotxt_msgs("user_key","app_key",text)

打掉注释，改一下follow5那行即可

4. 老方法上传

5. 去follow5那把同步功能选几个想要的开了
